REST API

This project aims at building an inventory tracking system for ventilators that allows authorized hospital users to change details of the available inventory and tag them as it as “occupied”, “available” and “in-maintenance”. These three status can also be updated using a secure API for real time usage. The application should have the ability to also add any ventilator to a specific hospital and search ventilators which are available.

With this API we can

  1. Read Hospital and Ventilator Details
  2. Search Ventilators by status and Hospital name
  3. Search Hospital by name
  4. Update Ventilators Detatils
  5. Add Ventilators
  6. Delete Ventilator by Ventilator ID

Technologies and Tools I used

  1. For storing Hospital and Ventilator Details I used Mongodb
  2. Brower only support GET and POST requests But I want to perform GET,POST,PATCH,DELETE so I used Postman
  3. Node.js for JavaScript run time Environnment
  4. Express.js
  5. MongoClient module used to interact with MongoDb
  6. Nodemon is used instead of node to run code, so process will automatically restart when we make any code changes
